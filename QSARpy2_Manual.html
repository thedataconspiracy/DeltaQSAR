QSARpy 2.1

QSARpy is a knowledge discovery suite for automatic QSAR/QSPR rules induction.

QSARpy Extractor:

 Input: Training set | Output: Model

Given an input dataset, QSARpy Extractor can extract a model of the endpoint of interest as a set of structural rules.
Training structures are broken down into fragments, then compared focusing on correlations between differential features. When there are statistical evidences that a structural variation (the presence/absence of a fragment) is related to a specific endpoint variation, then that fragment is recorded as a modulator.

The final model consists of the structures in the training set (molecular structures with an experimental endpoint value) and the modulator set (substructures known to cause predictable endpoint modulations when bonded to another structure). The prediction range of such model is extremely wide: it covers all the structures that can be derived starting from a training structure and subsequently adding or removing an indefinite number of modulator substructures. Modulators can be further organized into clusters to make it more readable.

QSARpy Predictor:

 Input: Test set + Model | Output: Predictions

Once extracted a model, QSARpy Predictor can use it to predict the endpoint of new structures from a test set.
When a structure of the test set can be perfectly derived starting from a structure of the training set and subsequently adding or removing modulator substructures, then a prediction can be calculated by modulating the relative experimental endpoint value.

This way, a prediction is triggered only when the model can account for the testing structure as a whole (no partial matches or read-across). Therefore, QSARpy predictions are thoroughly transparent and backed up by evidences from the training set. To get good performances, training data should be large enough [1].

[1] a few hundreds of molecules may be not enough to get sufficient prediction coverage

2.1 Fragmentation

To start a new session, load a training set from the file menu: File > Load Training set…

SET TRAINING SET HEADERS

Specify the required (∗) headers in the input dataset:

  • ID: CAS number or any other ID suitable to identify the molecular structure
  • SMILES: structural line notation  (CSV file format only)

The ID field is optional; however, models extracted by QSARpy refer to the training structures. Setting an appropriate structural identifier can facilitate reading the rationale behind predictions. Otherwise, the index of the structure in the input dataset will be used instead.

❒ Preprocess outliers: By ticking this box, input structures are prescreened either before training or testing. Outliers of the training set [1] are excluded from the learning process (no modulators extracted) and their structures are integrated in the model to early identify potential outliers in the test set (see Appendix B).

Fragmentation parameters...

You can control the computational load (time and memory) by restricting the depth of the fragmentation:

  • Fragmentation depth: the max number of bond breakings per molecule (i.e., the structures are broken down to max  fragmentation depth + 1  fragments). Since the space of possibilities has factorial growth [2], be careful: higher values can raise a memory error on large datasets ( > 10k ).
    ( Default = 3 )

  • SPEED UP: the fragmentation of the larger 5% structures [3] can take more than half of the resources. You can force a shallow fragmentation (fragmentation depth = 1) on just those structures by ticking this box.
    ( Default = True )

2.2 Extraction

The training structures and their fragments are analyzed. If a small fragment provides enough observations on how its presence/absence can affect the endpoint, and such endpoint variations are consistent throughout the observations, then that fragment is recorded as modulator: a substructure accountable for a constant shift of the endpoint of the structures to which it is bonded. Such endpoint shift is calculated as the mean of the sample of the related endpoint variations observed in the training set.

SET MAX ERROR THRESHOLD:

Given a fragment, the lower the deviation in the sample of endpoint variations, the higher the accuracy as potential modulator.
Set a maximum MAD [1] value: only fragments within the threshold will be retained. Too strict values can increase the number of unpredicted structures. Practically, this value is also a measure of the expected error of the resulting modulator [2].

Slide, spin or digit a value  ( Default = MAD training set )

Extraction parameters…

You can control the number of modulator candidates by constraining the fragment attributes:

  • MIN sample size: the minimum required number of (independent) differential observations for the given fragment in the training set
    ( Default = 4 )

  • MAX atoms: the max number of atoms
    ( Default = 8 )

  • Deep learning: include fragments from the deepest levels of fragmentation [3]
    ( Default = False )

It's a tradeoff: relaxing the constraints will increase the number of modulators and generally improve prediction capabilities; on the other hand, too many and excessively specific modulators would overfit the model.

2.3 Clustering

Enabled only if Enable CLUSTERED VIEW is checked in the previous tab.

The model can be rationalized (omitting redundancies and merging modulators) to give to the user a more readable view of the modulators. This process involves a minor review of the shift values of affected modulators, within certain margins defined by the user.

SET MODEL TOLERANCE:

Groups of related modulators may be clustered together around a common shift value, if such adjustment lays within a margin of tolerance set by the user. Set a value such that a variation within the limits may be considered irrelevant for the given endpoint.
Practically, model tolerance may be interpreted as the experimental error in the input data.

Slide, spin or digit a value  ( Default = max error threshold / 4 )

CLUSTER

Rationalize the model and print out a clustered view to the console. See Appendix A for details.

Console output

Each line displays a modulator cluster, with the common shift and the list of SMILES matched.
Clusters may be further grouped together if they share the same "base structure" (i.e., the SMILES without the starred bonds, showed in the first entry only). Different clusters of the same group are called variants. Groups with more than one variant are separated by a blank line.

Example:

CLUSTERED VIEW:


  group   shift   SMILES

  C       +1.76   *=C=*                 [1]  
          +0.36   *C*  *C  *=C  *C#*
            
  O       -0.53   *O  *O*  *=O
  N       -0.93   *=N  *N*  *N(*)*  *N  *#N
    
  ...

Clustered modulators are followed by the list of dummy modulators (i.e., substructures with no effect, clustered around shift = 0) and by the number of redundant modulators (i.e., modulators combination of other smaller modulators) omitted.

The new model is automatically validated against the training set and the statistics are prompted to the user.
The new model is ready to be saved in the Output menu.


Press    QSARpy Predictor >>    to import the model into QSARpy Predictor.

Appendixes

Appendix A

Model generalization

Removing redundancies - When a modulator structure is the union of the structures of other smaller modulators, and the difference in the relative shifts is less than the tolerance, then the bigger modulator is redundant. It is removed and the smaller and more generic modulator are showed instead.

Dummy modulators - Modulators related to a very small shift (absolute value less than the tolerance) are not really significant. They are kept just as ‘placeholders’ with no endpoint shift: they are substructures known for not altering the endpoint.

Clustering - When more modulators share the same structure and they differ only for the connection bonds noted by an asterisk in the SMILES (e.g., C∗ and C=∗), if their endpoint shifts are in a range of twice the tolerance, they are clustered together in a single meta-modulator that matches with all the SMILES. The endpoint shift of such meta-modulator is an average [1] of the single shifts.

[1] arithmetic mean weighted on the sample size: since it can differ from the middle point of the cluster, in this case case the new shift may lay slightly outside the margins of tolerance

Appendix B

Prediction


(preprocessing) - Prediction by Identity

Testing structures that are identical to any training structures are predicted by the relative endpoint values in the training set.

(preprocessing) - Prediction by Similarity

Enabled only if Preprocess outliers is enabled

Testing structures that are strictly similar [1] to any outlier structures are predicted by an average [2] of their endpoint values in the training set.
Unlike standard QSARpy prediction methods where the whole structure must be matched, outlier predictions -if enabled- are the only case of read-across.

Prediction by Factorization

For every testing structure Stest such that it can be derived starting from a training structure Strain by adding/removing any number n of modulator substructural patterns M1, M2, ..., Mn , then a prediction is calculated starting from the experimental Strain endpoint and modulating its value by adding/subtracting the modulator shifts:

 
    IF      Stest  =  Strain  ±  M1  ±  …  ±  Mn

    THEN    prediction(Stest)  =  endpoint(Strain)  ±  shift(M1)  ±  …  ±  shift(Mn)

[1] identical fingerprint AND atom number ± 10%
[2] median value

Appendix C

Exporting a Database


All the knowledge on the endpoint collected by the differential approach of QSARpy can be exported in a database. The output is a SQLite database of fragments paired with the observations of their effect on the endpoint in the training set.

There are 2 ways to export a database:

  • model-less: just load a training set and hit the shortcut button  Quick Export DB...  at the bottom-left of the Fragmentation tab
  • model-based: extract a model first, then select Export Database... [1] in the Output menu

A model-less database contains the most external fragments only (single asterisk in the SMILES), and it's obtained by direct comparison between training structures. A model-based database contains deeper fragments (depending on the fragmentation depth of the training set), but the effect of these fragments on the endpoint is derived with the help of deducted modulators.

Endpoint information

Add some information about the endpoint:

  • Name: the endpoint name

  • Unit: the unit of measurement

  • Colormap: if the endpoint range has a "good" (GREEN) and a "bad" (RED) side, choose the color mapping accordingly. Otherwise, the default color mapping will be used.

The output file can be queried within the Δ-QSAR environment by a new tool: Δ-QSAR Query (DeltaQSAR_query.exe), a search engine for molecular fragments and their effect on different endpoints.

[1] disabled if Deep learning is enabled (Extraction > Extraction parameters... > Deep learning)
